Authentication Test Harness

The Spiffy UI framework provides the client-side portion of tokenized identity. This sample contains a simple authentication server, but you will want to implement an authentication server which works for your application. This page gives you a chance to test your authentication server and make sure it will handle login and logout from the Spiffy UI framework.

This test will make the REST calls to authentication and get a token from the specified server. You can specify any server location which hosts an authentication server compatible with Spiffy UI.

The framework can only handle and show you some of the error messages from the server. To see all errors, enable the Firebug console or the Google Chrome developer tools console.
