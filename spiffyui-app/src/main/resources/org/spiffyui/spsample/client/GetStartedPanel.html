Get Started With Spiffy UI

So you're ready to try out Spiffy UI? Good for you! Right now you're looking at the Spiffy UI application where we show off what the framework can do. When you're ready to get started: create your own project, check out Hello Spiffy, or add Spiffy UI to your own build.

Get Hello Spiffy

Hello Spiffy is the Hello World! of Spiffy UI. It shows you everything you need to get Spiffy UI up and running, create a simple form, and make a REST call. The Hello Spiffy samples and many others are available at the Spiffy UI GitHub page.

Build your own

When you're ready to build your own project, you can start by copying either of the Hello Spiffy projects. You can also create your own project or adapt any existing GWT project to use Spiffy UI. Spiffy UI is just three simple steps away.

1. Add the Spiffy project dependency

You can have Spiffy automatically downloaded through a Nexus server, or you can manually download JARs from our downloads page. For automatic downloading, the steps are slightly different for Maven and Ant.

Apache Maven

Add the following to the pom.xml file for your project. 

   <dependency>
        <groupId>org.spiffyui</groupId>
        <artifactId>spiffyui</artifactId>
        <version>0.7.6</version>
    </dependency>

Apache Ant with Ivy

Then add the dependency to your ivy.xml file like this: 

    <dependency org="org.spiffyui" name="spiffyui" rev="0.7.6"/>

Manual dependencies

Build systems like Make, and others which do not (by default) support managed dependencies from central repositories, can easily use Spiffy UI by download the JARs from the Spiffy UI downloads page.

3rd-party dependencies

Spiffy UI has a small number of 3rd-party dependency JARs. You can add them to your project in the same way you added Spiffy UI. The dependencies are: 

    <dependency org="commons-logging" name="commons-logging" rev="1.1.1"/>
    <dependency org="commons-codec" name="commons-codec" rev="1.4"/>
    <dependency org="com.metaparadigm" name="json-rpc" rev="1.0"/>
    <dependency org="org.apache.httpcomponents" name="httpclient" rev="4.0.1"/>
    <dependency org="org.apache.httpcomponents" name="httpcore" rev="4.0.1"/>

2. Reference Spiffy in your HTML

Now the CSS and JavaScript files are included in your project when you build. The next step is to reference them in your HTML file. The Spiffy UI framework includes many JavaScript and CSS files, but they are all combined into two files for faster application loading. Reference this one file and the JQuery library in the head section of your HTML files like this: 

    <head>
    <script type="text/javascript" src="jquery.min.js"></script>
    <script type="text/javascript" src="spiffyui.min.js"></script>
    ...
    </head>

3. Import Spiffy in your GWT module

The last step is to import the Spiffy UI framework GWT module. Add the following line to your GWT module file: 

    <inherits name="org.spiffyui.spiffyui" />

That's all it takes.
